# 测试任务：毒蛇图腾核心机制验证

## 测试目标
验证毒蛇图腾的毒液攻击机制：每 5 秒对距离最远的 3 个敌人降下毒液，造成伤害并施加 3 层中毒。

## 测试等级覆盖
- [x] Lv.1 必须测试（验证基础机制）
- [ ] Lv.2/Lv.3 仅测试机制变化（纯数值提升无需测试）

## 关联机制
- Buff: 无
- Debuff: 中毒 (Poison) - 状态：✅ 已校验
- 图腾技能：毒液攻击

## 图腾分类确认
| 确认项 | 内容 | 来源 |
|--------|------|------|
| 图腾 ID | viper_totem | data/units/viper_totem.json |
| 所属阵营 | 毒蛇图腾 | GameDesign.md |

## 前置条件
- 图腾：毒蛇图腾 (viper_totem)
- 单位：1 个毒蛇 (viper) 用于观察
- 敌人：至少 5 个普通敌人（用于验证最远 3 个目标）
- 作弊 API 调用：`set_core_hp(99999)`, `spawn_enemy(5)`

## 验证点
| 验证项 | 预期日志/现象 | 验证方式 |
|--------|--------------|----------|
| 毒液攻击触发 | `[TOTEM] 毒蛇图腾触发毒液攻击，目标：[enemy1, enemy2, enemy3]` | 日志分析 |
| 毒液伤害 | `[TOTEM_DAMAGE] 毒液对敌人{ID}造成{伤害值}伤害` | 日志分析 |
| 中毒层数 | `[TOTEM_EFFECT] 敌人{ID}中毒层数：3` | 日志分析 |
| 目标选择 | 选择距离最远的 3 个敌人 | 日志分析/行为观察 |
| 触发频率 | 每 5 秒触发一次 | 时间间隔观察 |

## 场景构造步骤
1. 选择毒蛇图腾开局
2. 调用作弊 API `set_core_hp(99999)` 构造高血量核心
3. 购买并放置毒蛇单位
4. 召唤 5 个以上敌人，分散在不同距离
5. 观察毒液攻击触发情况，记录日志
6. 验证中毒层数是否正确叠加

## 依赖排除项（已校验，无需重复测试）
- 中毒 Debuff: 已在其他测试中校验

## 备注
- 需要验证目标选择逻辑是否正确（最远 3 个敌人）
- 注意观察毒液攻击是否有可见效果
