# 测试任务：鹰图腾核心机制验证 (复测 004)

## 测试目标
验证鹰图腾的暴击回响机制是否正确触发并通过日志输出。

## 测试等级覆盖
- [x] Lv.1 必须测试（验证基础机制）
- [ ] Lv.2/Lv.3 仅测试机制变化（纯数值提升无需测试）

## 关联机制
- Buff: 无
- Debuff: 无
- 单位技能：鹰图腾暴击回响
- 被动效果：暴击时 30% 概率触发回响

## 前置条件
- 图腾：鹰之图腾 (eagle_totem)
- 单位：风暴鹰 (storm_eagle) 或其他具有暴击能力的鹰图腾单位
- 敌人：基础敌人（用于触发攻击）
- 作弊 API 调用：set_god_mode(true) 或 set_core_hp(99999)

## 验证点
| 验证项 | 预期日志/现象 | 验证方式 |
|--------|--------------|----------|
| 鹰图腾选择 | "TotemSelected" 或 "eagle_totem" | 日志分析 |
| 单位部署 | 单位成功放置在网格上 | 日志分析 |
| 波次开始 | is_wave_active: true | 日志分析 |

| 回响伤害 | 回响伤害数值日志 | 日志分析 |

## 场景构造步骤
1. 启动游戏客户端，选择鹰图腾
2. 刷新商店直到出现风暴鹰 (storm_eagle)
3. 购买并部署风暴鹰到坐标 (-1, 0)
4. 调用 start_wave 开始波次
5. 观察战斗 60 秒，轮询 /observations 获取日志
6. 检测日志中的 [EAGLE_CRIT] 和 [TOTEM_EFFECT] 标记
