## 1. 问题概述

在对牛图腾、蝙蝠图腾、狼图腾进行全面测试时，虽然发现了代码级崩溃（CRASH-002、CRASH-WOLF-001），但日志中**完全缺少机制验证所需的关键信息**。

作为游戏策划，我无法通过黑盒观察确认以下核心机制是否正常运作：
- 图腾效果是否触发
- 伤害数值是否正确
- Buff/Debuff是否正常叠加
- 核心受击和治疗情况

## 4. 实施建议

### 4.1 代码埋点位置

| 文件 | 函数/位置 | 埋点内容 |
|-----|----------|---------|
| `MechanicBatTotem.gd` | `_on_totem_attack()` | 图腾触发日志、目标选择日志、流血施加日志 |
| `MechanicWolfTotem.gd` | `_on_totem_attack()` | 图腾触发日志、魂魄加成计算日志 |
| `MechanicWolfTotem.gd` | `on_enemy_died()` | 敌人阵亡魂魄获取日志 |
| `MechanicCowTotem.gd` | `on_core_damaged()` | 受伤充能累积日志 |
| `MechanicCowTotem.gd` | `trigger_counter()` | 全屏反击触发日志、伤害日志 |
| `BaseUnit.gd` | `deal_damage()` | 单位攻击伤害日志 |
| `Core.gd` | `take_damage()` | 核心受击日志 |
| `Core.gd` | `heal()` | 核心治疗日志 |

### 4.2 日志级别建议

- **ERROR**: 代码错误、异常状态
- **WARN**: 机制异常、数值异常
- **INFO**: 机制触发、状态变化（策划主要关注此级别）
- **DEBUG**: 详细计算过程、调试用

### 4.3 配置开关

建议在 `game_data.json` 中添加配置项：
```json
{
  "log_config": {
    "enable_totem_logs": true,
    "enable_damage_logs": true,
    "enable_buff_logs": true,
    "log_level": "INFO"
  }
}
```

## 未来扩展预判

### 6.1 机制扩展方向

基于当前图腾设计，预判未来可能扩展的机制：

1. **图腾组合技** - 多个图腾同时选择时的协同效果
   - 示例：蝙蝠+狼 = 流血敌人死亡时额外获得魂魄
   - 日志需求：组合技触发日志、协同效果计算日志

2. **图腾升级系统** - 图腾效果随游戏进程增强
   - 示例：狼图腾攻击频率从5秒提升到3秒
   - 日志需求：图腾升级日志、属性变化日志

3. **图腾专属遗物** - 增强特定图腾效果的遗物
   - 示例："血月"遗物增加蝙蝠图腾流血层数
   - 日志需求：遗物效果触发日志、效果叠加日志

4. **环境互动机制** - 图腾与地图元素的互动
   - 示例：牛图腾在岩石地形上减伤增加
   - 日志需求：环境检测日志、条件满足日志

### 6.2 日志系统扩展建议

建议技术总监在设计日志系统时预留以下扩展点：
