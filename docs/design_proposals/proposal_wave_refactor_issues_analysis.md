## 3. 问题详细分析

### 3.1 问题1: Shop.gd 信号参数不匹配

**日志**: `ai_session_cow_totem_20260302_155153.log`
**时间**: 15:52:04.062
**错误**:
```
ERROR: Error calling from signal 'wave_started' to callable:
'Control(Shop.gd)::on_wave_started': Method expected 0 argument(s), but called with 3.
```

**黑盒分析**:
- WAVE-REFACTOR-001 后，`wave_started` 信号发射了3个参数
- 但 `Shop.gd` 的 `on_wave_started` 方法定义不接受参数
- 这是重构后信号签名不一致导致的

**已验证机制** (崩溃前):
- ✅ 牛图腾选择正常
- ✅ 商店阵营过滤正常 (显示牛阵营单位: torch, cow_golem, sunflower, cauldron)
- ❌ 单位购买异常: 意图购买 iron_turtle，实际购买 meat
- ✅ 核心血量计算正常

---

**日志**: `ai_session_cow_totem_20260302_155657.log`

**黑盒分析**:

**已验证机制** (崩溃前):

- ✅ 商店阵营过滤正常
- ✅ 单位购买部署正常 (plant 部署到 (1,0))
- ✅ 核心血量计算正常

---

### 3.3 问题3: CombatManager 属性访问错误

**日志**: `ai_session_cow_totem_20260302_171744.log`
**时间**: 17:17:55.439
**错误**:
```
SCRIPT ERROR: Invalid access to property or key 'total_enemies_for_wave'
on a base object of type 'Node2D (CombatManager.gd)'.
```

**黑盒分析**:
- WAVE-REFACTOR-001 重构时，`total_enemies_for_wave` 属性被移除或重命名
- 但某些代码仍在尝试访问该属性
- 这是重构后的代码不一致问题

**已验证机制** (崩溃前):
- ✅ 牛图腾选择正常
- ✅ 商店阵营过滤正常 (显示牛阵营单位: rock_armor_cow, rabbit, rage_bear, bee)
- ❌ 单位购买异常: 意图购买 iron_turtle，实际购买 yak_guardian
- ✅ 核心血量计算正常 (500.0 → 1000.0)

## 5. 改进建议

### 5.1 紧急修复建议 (P0)

1. **修复 Shop.gd 信号参数**
   - 更新 `on_wave_started` 方法签名，接受3个参数
   - 或修改信号发射代码，不传递参数

2. **修复 CombatManager 属性访问**
   - 找到访问 `total_enemies_for_wave` 的代码
   - 更新为正确的属性名或访问方式

3. **调查 Parameter "t" is null**
   - 检查波次启动时的时序问题
   - 增加防御性编程保护

### 5.2 重构质量改进建议 (P1)

1. **重构后全面回归测试**
   - WAVE-REFACTOR-001 涉及20+文件修改
   - 需要全面测试所有波次相关功能

2. **信号签名一致性检查**
   - 建立信号发射和处理方法的签名检查机制
   - 使用 Godot 的强类型信号连接

3. **属性访问安全**
   - 使用 `has_method()` 和 `get()` 进行安全访问
   - 增加属性存在性检查
